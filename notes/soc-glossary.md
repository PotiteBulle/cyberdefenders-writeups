# SOC Glossary

This glossary contains common terms used in CyberDefenders labs and SOC investigations.

## Alert

A notification generated by a detection system such as a SIEM, EDR, IDS or sandbox.

## Artifact

A file, log, memory dump, PCAP or any other object provided for investigation.

## Beaconing

Repeated communication from a compromised host to an external server, often used by malware or C2 frameworks.

## Blue Team

The defensive side of cybersecurity focused on detection, monitoring, incident response and hardening.

## C2

Command and Control. Infrastructure used by malware or attackers to communicate with compromised systems.

## Credential Access

MITRE ATT&CK tactic involving techniques used to steal credentials such as passwords, hashes, tokens or browser-stored secrets.

## DFIR

Digital Forensics and Incident Response. The practice of investigating incidents and responding to security events.

## DLL

Dynamic Link Library. A Windows library file that can be loaded by programs.

## EDR

Endpoint Detection and Response. A security tool that monitors and responds to endpoint activity.

## Exfiltration

The unauthorized transfer of data from a system to an external destination.

## False Positive

An alert or detection that appears malicious but is actually benign.

## Hash

A cryptographic fingerprint of a file or data.

Common hash types:

- MD5
- SHA1
- SHA256

## IOC

Indicator of Compromise. A technical indicator that may suggest malicious activity.

Examples:

- hash
- IP address
- domain
- URL
- filename
- registry key
- mutex

## Initial Access

MITRE ATT&CK tactic describing how an attacker first gains access to a system.

## Malware Triage

Initial analysis of a suspicious file to understand its type, behavior and risk.

## MITRE ATT&CK

A knowledge base of adversary tactics, techniques and procedures.

## PCAP

Packet capture file containing network traffic.

## Persistence

Techniques used by attackers or malware to remain active after reboot or user logout.

## Phishing

A social engineering technique that uses deceptive messages to trick users into opening malicious links or attachments.

## Process Tree

A view showing parent and child processes created during execution.

## Sandbox

A controlled environment used to safely execute and observe suspicious files.

## SIEM

Security Information and Event Management. A platform used to collect, correlate and analyze security logs.

## SOC

Security Operations Center. A team or environment dedicated to monitoring, detecting and responding to cybersecurity incidents.

## Stealer

A type of malware designed to steal information such as browser passwords, cookies, autofill data, wallets or application credentials.

## Threat Hunting

Proactive search for suspicious or malicious activity in an environment.

## Threat Intelligence

Information about threats, malware, infrastructure, adversary behavior and indicators.

## TTP

Tactics, Techniques and Procedures. A way to describe adversary behavior.

## YARA

A rule-based tool used to identify files or malware families based on patterns.

## Sigma

A generic detection rule format that can be converted into SIEM-specific queries.
